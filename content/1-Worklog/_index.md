Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [AWS Account Setup, Discovery Tasks, and VPC Fundamentals](1.1-week1/)

**Week 2:** [Core AWS Services: IAM, VPC Setup, and EC2 Management](1.2-week2/)

**Week 3:** [IAM Roles, Cloud9, S3 Static Hosting, and RDS Database](1.3-week3/)

**Week 4:** [Amazon Lightsail Containers and EC2 Auto Scaling](1.4-week4/)

**Week 5:** [Monitoring with CloudWatch and Hybrid DNS with Route 53](1.5-week5/)

**Week 6:** [AWS CLI, DynamoDB NoSQL, and ElastiCache Acceleration](1.6-week6/)

**Week 7:** [CloudFront CDN, Edge Computing, and Draft AWS Architecture](1.7-week7/)

**Week 8:** [AWS Managed Microsoft AD, Grafana Dashboard, and Architecture Refinement](1.8-week8/)

**Week 9:** [Granular Access Control with Resource Tags and Architecture Integration](1.9-week9/)

**Week 10:** [Finalize AWS Architecture and UI/UX Optimization](1.10-week10/)

**Week 11:** [AWS Cost Optimization, Risk Review, and System Hardening](1.11-week11/)

**Week 12:** [System Testing, Project Handover, and Final Review](1.12-week12/)
